**ACT AS:** Senior Frontend Architect & UI/UX Engineer.
**PROJECT:** "Bari.Trekking" - A community trekking platform (Landing Page + Registration Flow).
**CONTEXT:** We are building a high-performance, aesthetically pleasing SPA for an outdoor community in Patagonia. The user is a Semi-Senior dev, so code must be clean, modular, and strictly typed.

---

### 1. TECH STACK (STRICT)
- **Core:** React 18.2+, TypeScript, Vite 5.4+ (Scaffold with `npm create vite@latest`).
- **Styling:** TailwindCSS 3.4+ (Mobile-first, Custom Theme).
- **Animation:** Framer Motion (for micro-interactions and smooth reveals).
- **Forms:** React Hook Form + Zod (Schema validation).
- **Routing:** React Router DOM v6+.
- **Icons:** Lucide React.
- **Performance:** Lighthouse-first approach (Lazy loading, semantic HTML, accessible contrast).

---

### 2. FILE STRUCTURE (Modular Architecture)
Follow this exact structure based on "Estudio CILS" standards:
src/ ├── components/ │ ├── ui/ # Primitives (Button, Card, Input, Accordion) │ ├── layout/ # Navbar, Footer, Container │ └── sections/ # Hero, Pricing, LegalRegistration, FAQ ├── hooks/ # useScroll, useFormLogic ├── lib/ # utils.ts (clsx/tailwind-merge), schemas.ts (Zod) ├── styles/ # index.css (Tailwind directives) └── App.tsx # Main composition

### 4. CORE FEATURES TO IMPLEMENT

- Full viewport height (`min-h-screen`).
- Background image (placeholder mountain URL) with a gradient overlay (`from-bari-dark/80` to transparent).
- **Content:** Title "COMPARTIENDO SENDEROS", Subtitle about autonomy, and two buttons ("Unirme" & "Protocolo").

#### B. Pricing Section (The "Plans")
- Grid layout with 3 cards:
  1. **Sendero (Free):** Minimalist, gray border.

#### C. Registration & Legal Flow (Critical UX)
- **Layout:** Split screen (Desktop: 2 cols, Mobile: Stacked).
- **Left Col (Legal):**
  - A component displaying the Terms & Conditions.
  - **Requirement:** Must be a scrollable container (`max-h-[400px] overflow-y-auto`).
  - **Content:** Summarize the provided "Términos y Condiciones" (Autonomy, Risk Assumption, No Guide Service).
- **Right Col (Form):**
  - Fields: Name, Last Name, Email.
  - **Validation:** Use Zod. Email must be valid.
  - **The "Handshake":** A Checkbox that is *only* clickable if the user has interacted with the legal text (or just ensure it validates "I accept T&C and declare physical fitness").
  - **Submit:** Button text "Ir al Pago" (Mockup function).

#### D. FAQ Section
- Simple Accordion component using Framer Motion for `AnimatePresence` (expand/collapse).
